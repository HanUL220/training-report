# PJT2: TEAM SOCCER (팀 프로젝트)

- **진행 기간:** 2024.08.21 ~ 2024.08.30
- **사용 기술:** React, Spring Boot, MySQL, Docker, Flyway, Git Action

## 프로젝트 개요
지역 축구팀 홍보와 선수 모집을 돕는 온라인 플랫폼.

## 주요 기능
1. 축구 팀 게시물 CRUD  
2. 축구 팀 가입 신청 게시물 CRUD  
3. 회원가입 및 JWT 인증 처리  
4. 검색 기능  
5. Docker, Docker Compose, Git Action을 활용한 CICD 파이프라인

## 트러블슈팅
1. **보안 강화**
   - **문제:** JWT 토큰의 보안성 강화 필요  
   - **해결:** Access Token + Refresh Token 병행 사용

2. **인프라 구축**
   - **문제:** 로컬 환경과 서버 환경을 효율적으로 관리해야 함  
   - **해결:** Docker Compose로 환경 구성 분리 (dev/stage/prod)

3. **데이터베이스 스키마 버전 관리**
   - **문제:** 개발 진행 중 DB 스키마 변경이 잦아 롤백이 어려움  
   - **해결:** Flyway 도입하여 각 버전별 마이그레이션 및 롤백 절차 확립

## 느낀 점
- **팀 협업의 중요성**: 여러 명이 동시에 코드를 수정하고, Git Action으로 CICD를 적용하면서 **체계적인 협업**을 위한 Git/GitHub 활용법을 더 깊이 배웠습니다.
- **인프라 및 배포 프로세스**: Docker와 Docker Compose를 통해 개발/배포 환경을 분리하고, Flyway로 DB 스키마 버전 관리를 체험하면서 **실무 환경 구성**이 얼마나 중요한지 절실히 느꼈습니다.
- **보안 강화**: JWT 토큰을 통한 인증을 구현하고, Access Token + Refresh Token 전략을 사용함으로써 **보안적인 요소**(세션 만료, 토큰 유효성 검사 등)를 실제 프로젝트에 적용해볼 수 있었습니다.
- **Agile 방식 적용**: 짧은 기간 동안 프로젝트를 진행하는 과정에서 스크럼 미팅, 칸반 보드 등을 사용하며 **유연한 태스크 관리**가 결과물의 질과 속도에 큰 영향을 준다는 사실을 알게 되었습니다.
